The whole keyspace,

rihawk is a streaming Riak client. Hand it buckets and keys — single values or arrays — and it walks every combination as one object-mode stream, prefetching the next batch while you process the last. Written in TypeScript, shipped with declarations, one runtime dependency.

§01 — six keys, two buckets, one stream

scout.mjs 
import rihawk from 'rihawk';

const client = rihawk({ connectionString: '127.0.0.1:8087' });

client.get(['nfl_team', 'mlb_team'], ['CHI', 'MIA', 'SD'], { r: 2 }, { concurrent: 8 })
  .on('data', (record) => console.log(record.bucket, record.key))
  .on('error', (error) => console.error(error.bucket, error.key)) // knows what failed
  .on('end', () => client.end());

Arrays expand. Two buckets × three keys becomes six requests, walked last-dimension-fastest and emitted as one ordered stream of records.

Latency overlaps. While your handler runs, the next batch is already on the wire — bounded read-ahead, never more than one batch beyond demand.

Failure is addressed. Errors carry their coordinate (error.bucket, error.key) and the stream keeps going. Not-found keys are simply skipped.

§02 — equipment

01 / CARTESIAN STREAMS

Every method takes single values or arrays for bucket, key, value, index, and op — the full product streams back as one readable.

02 / READ-AHEAD PIPELINE

The next batch is prefetched while the consumer drains. Tune concurrent and highWaterMark per call.

03 / CRDTS

Counters, sets, and maps via getCrdt / putCrdt, with an updateCounter shorthand.

04 / ABORT & BACKPRESSURE

Pass an AbortSignal to cancel mid-flight. Native object-mode backpressure; destroy() stops issuing requests.

05 / TYPESCRIPT

Written in strict TypeScript. CJS and ESM entry points each ship correct declarations — validated by publint and attw in CI.

06 / LIGHT FOOTPRINT

One runtime dependency (no-riak), zero known vulnerabilities, pooled TLS-capable connections.
